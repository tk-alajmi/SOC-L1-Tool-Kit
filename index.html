🔌 Port reference — quick cheat sheet
Port alone never makes something malicious — context is everything. Port 3389 from IT to a server = normal. Port 3389 from Finance to an external IP at 2am = escalate.
PortServiceRiskSOC notes
21FTPWatchUnencrypted file transfer. External FTP = possible data exfiltration.
22SSHWatchLegitimate for IT. External SSH from non-IT user = investigate.
23TelnetHighUnencrypted legacy protocol. Almost never legitimate in modern environments.
25SMTPWatchEmail. Unusual outbound SMTP from workstation = malware sending spam.
53DNSWatchNormal in small amounts. Hundreds per minute = DNS tunneling.
80HTTPNormalStandard web. Watch for HTTP to raw IPs (not domains) — suspicious.
135WMI / RPCWatchWindows management. Between workstations = lateral movement indicator.
137–139NetBIOSWatchWindows file sharing. Scanning these ports = network reconnaissance.
389LDAPWatchActive Directory queries. Unusual volume = AD enumeration by attacker.
443HTTPSNormalEncrypted web. C2 can hide in HTTPS — check if destination domain is known.
445SMBWatchWindows file sharing. WannaCry used SMB. Lateral movement indicator.
1337Leet / MalwareHighNo legitimate service. Almost always malware C2.
3306MySQLWatchDatabase. External exposure = critical misconfiguration or data theft.
3389RDPWatchRemote Desktop. External RDP without VPN = major red flag.
4444MetasploitHighDefault Metasploit reverse shell. Almost always malicious.
4545 / 5555Malware C2HighCommon C2 ports. No legitimate service uses these.
6666–6667IRC / BotnetHighOld IRC / botnet C2. Rarely legitimate in enterprise environments.
8080 / 8443Alt HTTP/SWatchAlternative web ports. C2 uses these to mimic web traffic.
9001TorHighTor relay port. Internal machine using Tor = major red flag.
